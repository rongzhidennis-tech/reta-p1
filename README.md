# RetaP1

A learning project on the way to **Reta** — a macOS menu-bar app that helps
college students stay present in lectures through active recall.

This repository is that journey, built phase by phase to learn Swift, SwiftUI,
and the Xcode/Git workflow from the ground up. Phases build on each other in
this one app, and each completed phase is tagged (e.g. `v0.1-phase1`), so any
earlier state is one `git checkout` away.

## What this app does (so far)

Phases 1–3 are complete. Reta lives in the menu bar (a brain icon, no Dock
icon) and, on "Start listening", captures live microphone audio and
transcribes it in real time on-device. It detects seams — sustained pauses
that end a spoken stretch — and at each seam it seals the transcript into a
paragraph, restarts recognition (avoiding long-session decay), and shows a
small floating card at the top-right (over full-screen apps, without stealing
keyboard focus) asking a short retrieval question about what was just said.
Questions are generated by Claude via a small Cloudflare Worker; if the
network is slow (>3 s), down, or the server errors, an on-device template
question appears instead — the card never fails to appear. Stop ends the
session cleanly.

## Privacy — what leaves the Mac

- **Audio never leaves the Mac.** Capture and speech-to-text are entirely
  on-device (`SFSpeechRecognizer` with `requiresOnDeviceRecognition`).
- **Transcript text does leave the Mac for question generation.** At each
  seam, the sealed paragraph (the last spoken stretch, as text) is sent over
  HTTPS to a personal Cloudflare Worker, which forwards it to the Anthropic
  API and returns one question. Nothing is stored by the Worker.
- Seam detection, the transcript display, and all fallback questions are
  computed on-device.

## How it works, in one line

The microphone feeds an `AVAudioEngine` tap; each audio buffer is appended to
an on-device `SFSpeechRecognizer` request whose partial results update an
observable `transcript` that SwiftUI redraws live; a seam fires when the
transcript stops growing for 2 seconds; the sealed paragraph goes to the
`reta-prompts` Worker, and Claude's question (or the template fallback) shows
on a floating, non-activating `NSPanel`.

## Known limitations / roadmap

- The recall interaction is passive: the card shows a question and a
  "Got it" button. Phase 4 makes it real active recall — reveal-answer and
  self-rating.
- The pause length (2.0 s) is a constant in `AudioListener`; seams fire from
  the last recognized word, so cards can lag a beat behind the actual pause.
- Seams are pause-based; topic transitions without a pause are not detected
  (smarter, content-aware timing is future work).
- Question quality is tuned by editing the `INSTRUCTIONS` constant in
  `worker/src/index.js` and redeploying.
- If lecture halls defeat word-based detection, the next rung is a real
  voice-activity-detection model.

## Requirements

- macOS
- Xcode (version 26 or newer)
- For question generation: a deployed `worker/` (Cloudflare account +
  Anthropic API key stored as Worker secrets) and a local
  `RetaP1/Secrets.swift` (gitignored) with the Worker's shared token.
  Without these, the app still runs — cards fall back to template questions.

## How to run it

1. Open `RetaP1.xcodeproj` in Xcode.
2. Press **⌘R** (or the ▶ button) to build and run.
3. No window appears — look for the brain icon in the menu bar (top-right).
4. Click it, then "Start listening" and speak. First run: macOS asks for
   microphone and speech-recognition permission.
5. Pause ~2 seconds after saying something — the prompt card floats up at the
   top-right.
6. Quit with the popover's "Quit Reta" button.

## Project layout

```
RetaP1/
├── RetaP1.xcodeproj      Xcode's project database (build settings, file list)
├── RetaP1/               App source code
│   ├── RetaP1App.swift      App entry point — the MenuBarExtra scene
│   ├── ContentView.swift    The popover UI: transcript, Start/Stop, Quit
│   ├── AudioListener.swift  Mic capture, transcription, seam detection
│   ├── PromptCard.swift     The floating retrieval-prompt card (NSPanel)
│   ├── PromptService.swift  HTTPS client for the Worker (3 s timeout)
│   ├── PromptMaker.swift    On-device template questions (the fallback tier)
│   ├── Secrets.swift        Gitignored — shared token for the Worker
│   └── Assets.xcassets      Images, colors, and the app icon
└── worker/               Cloudflare Worker: paragraph in, Claude question out
    └── src/index.js         Auth check + Claude API call (key in a secret)
```

## The bigger vision (future phases)

Make the card a true recall beat (answer reveal + self-rating), persist
questions for after-class and spaced review, and make prompt timing
content-aware (topic transitions, not just pauses) — the hard core of the
product.
